MODELO RELACIONAL

SQL = Structured Query Language
DDL - linguagem de definição (ex. create database, create table, alter/drop table)
DML - linguageme de manipulação (ex. insert into, delete, update, truncate)
DQL - linguagem de solicitação (ex. select)
DCL - linguagem de controle
DTL - linguagem de transações
OBJETIVO - registrar instâncias separadas de coisas que têm características semelhantes
conjuntos de tabelas (que são conjuntos de registros)

TIPOS DE DADOS
    - numérico (inteiro, real e lógico)
    - data/tempo
    - literal (caractere, texto, binário e coleção)
    - espacial
constraint - regras para a criação de coisas dentro do banco
enum - define o que pode ser aceito

COMANDOS
    - drop - drop database/drop table
    - default character set utf8mb4
    - default collate utf8mb4_general_ci;
    - auto_increment
    - decimal(5, 2) - primeiro número: quantas casas, segundo número: quantas casas depois da vírgula
    - create/drop table if not exists
    - ALTER TABLE (para coluna) {
        add column
        drop column
        modify column (não permite renomear)
        change column - [nome antes] [nome novo] [constraint]
        rename to (para nome da tabela)
        add foreign key references
    }
    - SELECT {
        - order by [coluna] desc
        - order by [coluna1], [coluna2]
        - where [coluna] = 'x'
        - where [coluna] between 'x' and 'y'
        - where [coluna] in ('x', 'y')
        - where [coluna1] > 'x' and [coluna2] < 'y'
        - where [coluna] like 'x%'
        - where [coluna] like '%x%'
        - where nome like 'xy%x_';
        - where nome like 'x__y%'
        - select distinct - lista só que apenas uma vez cada ocorrencia
        - select count([coluna])
        - select count(*) from cursos
            where [coluna] > x
        - select max([coluna])
        - select [coluna1], min([coluna2])
        - select sum([coluna1])
        - select avg([coluna2])
        - ordem: select, where, group by, having, order
    }
    unique - nao deixa cadastrar mais de um daquele campo com o mesmo nome
    unsigned - sem sinal

        limit - limita quantas linhas seráo afetadas
        UPDATE - set where (limit)

chave estrangeira - chave primária de algum lugar que foi para outro (1, n - a do lado 1 vai para o n; n, n - o relacionamente vira atributo e os relacionamentos entre as entidades e a 3 tabela vira 1, n)
engine - maquina de criacao de tabela
ACID - regras de transação (ações do BDD): 
    - Atomicidade - a tarefa tem que ser feita por completo
    - Consistência - consistência antes e depois da transição
    - Isolamento - se duas transações ocorrerem ao mesmo tempo uma não pode afetar a outra
    - Durabilidade - o dado tem que durar até o tempo desejado
MUL - chave múltipla (estrangeira)
não se pode apagar uma linha que tem chave estrangeira em algum ponto
join - aparece somente com as relações
select [tabela.coluna]
from [tabela1] (left/right, outer/inner)join [tabela2]
on [tabela2.colunax] = [tabela1.colunax]

git ≠ github
git 
    - software de controle de versão (VCS)
    - versionamento de software
    - versões
    - source code managment (SCM)
github 
    - repositório remoto
    - repositórios ilimitados
    - hospedagem de código-fonte
    - características de rede social
    - github pages integrado
    - colaboração
    - fork
    - proprietário
repositório central/linear - enviar pasta para repositório central (tem que estar conectado ao servidor) 
repositório distribuído - repositório local e repositório remoto
github -plataforma de rede social para programadores

PRINCIPAIS VANTAGENS
- controle de histórico
- trabalho em equipe
- ramificação do projeto
- segurança
- organização
commit - envia para repositório local
push - pegar do repositório local e enviar para o repositório remoto
pull - pegar do repositório remoto e enviar para o repositório local

ATALHOS
abrir vs code - Ctrl + Shift + A
push - Ctrl + p

ISSUES
fazer uma busca sobre a issue, evitar duplicar

MARKDOWN
    - tipo HTML 
    - usada em README, issues and pull request
    - *[texto]* - itálico
    - **[texto]** - negrito
    - preview para ver como vai ficar
    - # - h1
    - ## - h2
    - ### - h3
    - *** - faz uma linha
    - [número]. - lista numerada
    - * - lista demarcada
    - - [ ] - lista de tarefas (por x dentro dos colchetes)
    - máx imagem: 400/ 500 px
    - [frase] (link)
    - [ [tema 1] | [tema 2] | [tema 3]]
        ---|---|--- -  tabela
    - ` para por código
    - : [nome] - ára por emoji (github.com/ikatyang - nome dos emoji no repositorio)
    - marca-se usuário por @
    - quote reply - responde alguém 
    - > - quote

BRANCHES
    - master: ramificação principal/ oficial
    - origin: repositório remoto
    - branch: ramificação que não é a oficial
    - merge: passar a branch para a master
